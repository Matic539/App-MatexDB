    Repositorios

    Esta sección documenta la capa de Repositorios (SQL puro) que encapsula los SELECT/INSERT/UPDATE/DELETE contra PostgreSQL.

    repository.db

    Este módulo gestiona el pool de conexiones y el contexto transaccional:

    Manejo de conexiones PostgreSQL (capa Repository).

    Contiene un pool global de conexiones y un context manager get_conn que garantiza commit/rollback y la devolución segura al pool.

    get_conn()

    Context manager que entrega una conexión del pool.

    init_pool(minconn=1, maxconn=10)

    Inicializa el pool global de conexiones.

    Parameters:

    Name Type Description Default
    minconn int

    Conexiones mínimas a mantener.

    		 1
    maxconn int

    Conexiones máximas permitidas.

    		 10
    Side effects

    Crea la variable global _pool si aún no existe.

    Raises:

    Type Description
    Error

    Si la conexión a la BD falla.

    repository.producto_repo

    Operaciones CRUD sobre la entidad Producto y su precio.

    Operaciones CRUD sobre productos y precios.

    Esta capa NO contiene lógica de negocio; solo SQL y transformaciones mínimas a tipos dict / list.

    actualizar(id_prod, precio, stock)

    Actualiza precio y stock de un producto existente.

    crear(nombre, precio, stock)

    Crea un nuevo producto con su precio inicial.

    descontar_stock(id_prod, cantidad)

    Resta cantidad al stock del producto indicado.

    eliminar(id_prod)

    Elimina un producto (y su precio) de la BD.

    exportar_excel()

    Exporta el inventario a data/inventario_exportado.xlsx.

    Returns:

    Type Description
    str

    Ruta absoluta del archivo generado.

    listar(*, stock_bajo=False, sin_precio=False)

    Devuelve los productos filtrados.

    Parameters:

    Name Type Description Default
    stock_bajo bool

    Solo productos con stock ≤ 30.

    		 False
    sin_precio bool

    Solo productos sin precio asignado.

    		 False

    Returns:

    Type Description
    list[dict[str, Any]]

    Lista de dicts con id, nombre, precio y stock.

    obtener_precio(id_prod)

    Devuelve el precio neto de un producto (o 0 si no existe).

    obtener_productos()

    Lista id, nombre y stock de todos los productos.

    obtener_stock(id_prod)

    Devuelve el stock disponible de un producto.

    repository.ventas_repo

    Operaciones CRUD y consultas sobre Ventas y su detalle (ventas_producto).

    Operaciones CRUD y reportes de ventas.

    detalle(id_venta)

    Devuelve el detalle de productos vendidos en una venta.

    eliminar_venta(id_venta)

    Elimina una venta y restaura el stock de sus productos.

    exportar_excel()

    Exporta la tabla de ventas a data/ventas_exportadas.xlsx.

    Returns:

    Type Description
    str

    Ruta absoluta del archivo generado.

    insertar_venta(venta, items)

    Inserta cabecera y detalle de una venta.

    Parameters:

    Name Type Description Default
    venta Dict[str, Any]

    Dict con fecha, forma_pago, monto_total, total_productos.

    		 required
    items Sequence[Dict[str, Any]]

    Secuencia de dicts con id_producto, cantidad, monto_producto.

    		 required

    Returns:

    Type Description
    int

    id_venta generado.

    listar(rango=None)

    Lista ventas opcionalmente filtradas por rango de fechas.

    Parameters:

    Name Type Description Default
    rango tuple[str, str] | None

    Tupla (desde, hasta) en formato YYYY-MM-DD.

    		 None

    Returns:

    Type Description
    list[dict[str, Any]]

    Lista de dicts con id, fecha, forma, total_prod, monto.
